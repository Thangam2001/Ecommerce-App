E-Commerce MVP — Feature Document & Backend API Plan

🎯 MVP Philosophy
Build only what's needed to launch, sell, and operate. Cut everything that can wait for v2.

✅ MVP Modules (6 Core Only)
#ModuleWhy It's MVP Critical1AuthenticationEvery action depends on identity2Product & CategoryNothing to sell without this3CartCore shopping flow4Order ManagementComplete the purchase loop5PaymentRevenue — the whole point6Admin Panel (Basic)Manage products & orders

❌ Deferred to v2: Wishlist, Reviews, Coupons, Notifications, Analytics, Social Login, Variants, Shipping config


🔐 Module 1 — Authentication & Authorization

Features

Register with email + password
Email verification (OTP or link)
Login → returns JWT access token
Refresh token for session continuity
Forgot password / Reset password via email
Role-based access: ADMIN, USER, GUEST
Logout (invalidate token server-side)

API List
MethodEndpointAccessPurposePOST/api/auth/registerPublicRegister new userPOST/api/auth/verify-emailPublicVerify via OTP/tokenPOST/api/auth/loginPublicLogin, receive JWTPOST/api/auth/refresh-tokenPublicRenew access tokenPOST/api/auth/forgot-passwordPublicSend OTP to emailPOST/api/auth/reset-passwordPublicReset with OTP + new passwordPOST/api/auth/logoutUserInvalidate tokenGET/api/auth/meUserGet logged-in user info

🛍️ Module 2 — Product & Category Management
Features

Products have name, description, price, stock count, images, SKU
Single category per product (no nested categories for MVP)
Multiple images per product
Product status: Active / Inactive
Guest and user can browse & view products
Admin creates, edits, deletes, and manages status
Basic related products (same category)

Add to cart with stock validation
Update item quantity

📦 Module 4 — Order Management
Features

Place order from cart
Save delivery address at order time (no address book for MVP)
Order statuses: PENDING → CONFIRMED → SHIPPED → DELIVERED → CANCELLED
User can cancel before SHIPPED
Snapshot product price at time of purchase (price won't change with product edits)
Admin views and updates order status
User views own order history and detail

API List — User
MethodEndpointAccessPurposePOST/api/orders/placeUserPlace order from cartGET/api/ordersUserMy order historyGET/api/orders/{id}UserView order detailPUT/api/orders/{id}/cancelUserCancel order

💳 Module 5 — Payment
Features

Razorpay integration (best for India / INR)
Flow: Place Order → Initiate Payment → Gateway → Verify → Confirm Order
Payment statuses: PENDING, SUCCESS, FAILED, REFUNDED
Auto-confirm order on successful payment
Basic refund trigger by admin on cancellation
Payment webhook to handle async gateway callbacks

API List

🖥️ Module 6 — Admin Panel (Basic)
Features

Admin-only dashboard with minimal stats
Manage users (view, activate/deactivate)
All product and order management (covered in Modules 2 & 4)
Low stock alert view
No complex analytics for MVP — just counts
